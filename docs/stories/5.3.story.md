# Story 5.3: Revue de code et corrections ML Service

**Status:** Done

## Story

**As a** mainteneur du projet,  
**I want** appliquer les corrections issues de la revue de code du service ML,  
**so that** le code est plus robuste, sécurisé et maintenable.

## Acceptance Criteria

**Given** un rapport de revue de code (code-review-ml-service-2026-02-12.md)  
**When** les corrections sont appliquées  
**Then** un context manager async est disponible pour les connexions DB (`Database.connection(tenant_id)`)  
**And** les données d’accuracy sont validées (`_prepare_data` : colonnes date/quantity ou ValueError)  
**And** les réponses HTTP (chat, auth) n’exposent pas les détails d’exceptions techniques  
**And** le nombre de snapshots par tenant est limité (éviction FIFO, 500 max)  
**And** les origines CORS sont configurables via variable d’environnement  
**And** le comportement `daily_consumption == 0` du modèle baseline est documenté  
**And** les tests couvrent la validation des données et l’éviction des snapshots
