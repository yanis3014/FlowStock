# Story 8.2: Refonte du Layout partagé — Navigation et header en Tailwind

**Status:** Done

## Story

**As a** utilisateur,  
**I want** une barre de navigation et un header commun sur toutes les pages métier, stylés avec des classes utilitaires Tailwind,  
**so that** je navigue de façon cohérente et l'identité visuelle est unifiée sans CSS custom ni variables CSS natives.

## Acceptance Criteria

**Given** je suis sur une page métier (dashboard, stats, chat, formules, import, etc.)  
**When** la page est chargée  
**Then** un layout partagé affiche la barre de navigation (liens Dashboard, Stats, Prévisions, Chat, Formules, Ventes, Imports, Mouvements, Référentiels)  
**And** le header affiche le titre de la page courante et un lien Déconnexion (ou profil) si authentifié  
**And** la navigation et le header utilisent exclusivement des classes utilitaires Tailwind (remplacement du CSS custom)  
**And** les pages login et register n'affichent pas la nav applicative  
**And** les contrastes pour les badges d'alerte respectent ≥ 4,5:1 (WCAG AA)

## Tasks / Subtasks

- [x] Task 1 — Identifier et remplacer le CSS du layout
  - Repérer le layout partagé actuel (layout.js, include commun, ou duplication nav/header dans chaque HTML)
  - Supprimer les styles CSS custom associés à la navigation et au header (variables CSS ou feuilles dédiées)
  - Remplacer par des classes Tailwind : conteneur, flex/grid, couleurs (bg-primary, text-*), espacements (p-*, m-*), bordures, responsive (md:, lg:)

- [x] Task 2 — Navigation et header en Tailwind
  - Barre de navigation : flex, liens avec états hover/active utilisant la charte (primary, etc.)
  - Header : titre de page + lien Déconnexion/profil ; alignement et espacements en Tailwind
  - Conserver le comportement existant (liens actifs, mobile si menu burger) avec classes Tailwind uniquement

- [x] Task 3 — Exclusions et accessibilité
  - S'assurer que login.html et register.html n'affichent pas la nav applicative (aucun changement de comportement)
  - Vérifier les contrastes des éventuels badges d'alerte dans le layout (≥ 4,5:1 WCAG AA)
