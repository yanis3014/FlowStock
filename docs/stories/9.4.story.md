# Story 9.4: Tableaux et Filtres Intelligents

**Status:** Ready

## Story

**As a** gérant de PME,  
**I want** des tableaux triables et filtrables sans rechargement de page,  
**so that** je consulte et gère Fournisseurs, Emplacements et Ventes efficacement.

## Acceptance Criteria

**Given** les pages Fournisseurs, Emplacements et Ventes migrées en composants React  
**When** je consulte une de ces pages  
**Then** les données sont chargées automatiquement au montage  
**And** les tableaux sont triables par colonne (clic sur en-tête)  
**And** des filtres de recherche ou par critères sont disponibles sans rechargement  
**And** les composants utilisent Shadcn Table ou équivalent pour un rendu professionnel  
**And** la pagination ou le chargement progressif est géré pour les grandes listes  
**And** les actions (créer, modifier, supprimer) s'exécutent en contexte sans rechargement complet

## Tasks / Subtasks

- [ ] Task 1 — Page Fournisseurs (suppliers)
  - Composant React avec tableau des fournisseurs
  - Tri par colonne (nom, contact, produits associés, etc.)
  - Filtre de recherche par nom
  - Actions CRUD en modal ou inline
  - Chargement automatique au montage

- [ ] Task 2 — Page Emplacements (locations)
  - Composant React avec tableau des emplacements
  - Tri par colonne
  - Filtre de recherche
  - Actions CRUD
  - Chargement automatique au montage

- [ ] Task 3 — Page Ventes (sales)
  - Composant React avec tableau des ventes
  - Tri par date, produit, quantité
  - Filtres par date, produit
  - Pagination ou virtualisation pour grandes listes
  - Chargement automatique au montage

- [ ] Task 4 — Composants réutilisables
  - Extraire un composant DataTable générique (tri, filtres, pagination)
  - Utiliser Shadcn Table comme base
  - Harmoniser le style entre les trois pages
