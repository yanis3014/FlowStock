# Story 2.1: CRUD Stocks de Base

**Status:** Done

## Story

**As a** gérant de PME,  
**I want** créer, voir, modifier et supprimer mes stocks,  
**so that** je peux gérer mon inventaire de base.

## Acceptance Criteria

**Given** je suis un utilisateur authentifié  
**When** je crée un nouveau stock  
**Then** je peux saisir les informations produit (nom, description, SKU, quantité, unité, emplacement)  
**And** la validation des données fonctionne (quantité >= 0, champs obligatoires)  
**And** le stock est enregistré en base de données avec association au tenant  
**When** je consulte mes stocks  
**Then** je vois la liste de tous mes stocks avec leurs informations  
**When** je modifie un stock  
**Then** je peux mettre à jour toutes les informations  
**And** les modifications sont enregistrées  
**When** je supprime un stock  
**Then** je reçois une confirmation avant suppression  
**And** le stock est supprimé (soft delete) de la base de données  
**And** les messages de confirmation sont affichés pour toutes les actions  
**And** les erreurs sont gérées gracieusement (produit inexistant, validation échouée)  
**And** les tests unitaires et integration pour CRUD sont passants

### Data Models

- **locations** (PostgreSQL) : [Source: docs/database-schema.md#4-locations-table]  
  - Table support pour product.location_id ; gestion complète des emplacements dans Story 2.3.
- **suppliers** (PostgreSQL) : [Source: docs/database-schema.md#5-suppliers-table]  
  - Table support pour product.supplier_id ; gestion complète des fournisseurs dans Story 2.5.
- **products** (PostgreSQL) : [Source: docs/database-schema.md#6-products-table]  
  - Champs : sku, name, description, unit (product_unit), quantity, min_quantity, location_id, supplier_id, purchase_price, selling_price, lead_time_days, is_active. Soft delete via is_active. stock_status ("ok", "low", "critical") est dérivé (calculé côté API), pas stocké en BDD.

### API Specifications

- **GET /products** : [Source: docs/api-specifications.md#2-products-service]  
  - Query : page, limit, search, location_id, supplier_id, low_stock, sort, order. Réponse 200 avec data[] et pagination.
- **GET /products/:id** : 200 avec détail produit ; 404 si non trouvé.
- **POST /products** : body sku, name, description, unit, quantity, min_quantity, location_id, supplier_id, purchase_price, selling_price, lead_time_days. 201 Created ; 400 Bad Request ; 409 Conflict (SKU exists).
- **PUT /products/:id** : même body (tous optionnels). 200 OK ; 404 ; 400.
- **DELETE /products/:id** : 204 No Content ; 404.
