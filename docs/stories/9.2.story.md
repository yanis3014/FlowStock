# Story 9.2: Shell Applicatif Moderne (Layout Next.js)

**Status:** Ready

## Story

**As a** utilisateur,  
**I want** un layout racine unifié avec Sidebar rétractable et Header dynamique,  
**so that** je navigue de façon fluide sans rechargement de page et avec une identité visuelle professionnelle.

## Acceptance Criteria

**Given** l'application Next.js  
**When** une page métier est affichée  
**Then** un Layout racine inclut une Sidebar rétractable (desktop) et un Header avec titre de page et actions utilisateur  
**And** la Sidebar utilise Tailwind et Shadcn UI (composants Button, Sheet pour mobile)  
**And** la navigation entre Dashboard, Stats, Prévisions, Chat, Formules, Ventes, Imports, Mouvements, Référentiels ne recharge que le contenu central  
**And** les pages login et register ne présentent pas la Sidebar ni le Header applicatif  
**And** le layout est responsive (menu burger sur mobile, sidebar fixe sur desktop)  
**And** l'identité visuelle est cohérente avec la charte (primary, success, warning, error)

## Tasks / Subtasks

- [ ] Task 1 — Structure Layout racine Next.js
  - Créer un Layout principal (ex. `app/(app)/layout.tsx`) qui enveloppe les pages métier
  - Isoler les routes login et register dans un groupe sans ce layout (ex. `app/(auth)/layout.tsx`)

- [ ] Task 2 — Sidebar rétractable
  - Intégrer un composant Sidebar avec Shadcn UI (ou équivalent)
  - Liens vers : Dashboard, Stats, Prévisions, Chat, Formules, Ventes, Imports, Mouvements, Référentiels (Emplacements, Fournisseurs)
  - Bouton pour rétracter/étendre la Sidebar sur desktop
  - Sur mobile : menu burger ouvrant un Sheet/Drawer

- [ ] Task 3 — Header dynamique
  - Header avec titre de la page courante (dérivé de la route ou contexte)
  - Lien Déconnexion ou profil utilisateur
  - Style cohérent avec la charte Tailwind

- [ ] Task 4 — Navigation client-side
  - Utiliser `<Link>` de Next.js pour une navigation sans rechargement
  - Vérifier que le contenu central change sans recharger la Sidebar et le Header
  - Tester sur desktop et mobile (responsive)
