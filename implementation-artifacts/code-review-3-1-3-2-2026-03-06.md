## Findings Story 3.2

### CRITICAL

*Aucun.*

### HIGH

1. **Retour vers l’onboarding manquant**  
   - **Fichier :** `apps/web/src/app/(app)/import-stocks/page.tsx`  
   - **Problème :** Task 4.1 : « soit navigation vers `/import-stocks` (recommandé MVP), **avec retour possible vers l'onboarding** ». La page import-stocks propose uniquement « Retour aux stocks », pas de lien vers l’onboarding.  
   - **Correction :** Ajouter un paramètre `?from=onboarding` et afficher un lien « Retour à l’onboarding » quand l’utilisateur vient de l’onboarding.

2. **Preview limité à 10 lignes au lieu de 20**  
   - **Fichier :** `apps/web/src/app/(app)/import-stocks/page.tsx` ligne 341  
   - **Problème :** Task 2.1 : « tableau preview (20 lignes max renvoyées par l'API) ». L’API renvoie bien jusqu’à 20 lignes (`product-import.service.ts` ligne 285), mais l’UI fait `preview.sampleRows.slice(0, 10)`.  
   - **Correction :** Afficher les 20 lignes : `preview.sampleRows.slice(0, 20)` ou supprimer le slice.

3. **Retry CSRF incompatible avec FormData**  
   - **Fichier :** `apps/web/src/hooks/useApi.ts` lignes 74-94  
   - **Problème :** En cas de 403 CSRF, le retry ne s’applique que si `isBodyReplayable(options.body)` est vrai (body de type string). Pour `FormData`, le body n’est pas rejouable, donc pas de retry. Les POST `/products/import/preview` et `/products/import` en FormData peuvent échouer définitivement si le token CSRF a expiré.  
   - **Correction :** Pour FormData, soit récupérer un nouveau token CSRF et réessayer (sans rejouer le body si impossible), soit documenter la limitation et prévoir un message d’erreur explicite.

### MEDIUM

4. **Zone de drop non cliquable**  
   - **Fichier :** `apps/web/src/app/(app)/import-stocks/page.tsx` lignes 253-269  
   - **Problème :** La zone de drop ne déclenche le file picker qu’au clic sur le bouton « Parcourir ». Une UX courante est de rendre toute la zone cliquable.  
   - **Correction :** Ajouter `onClick={() => fileInputRef.current?.click()}` sur la div de drop.

5. **Message d’erreur location/supplier inconnu**  
   - **Référence :** Dev Notes : « L’UI doit l’expliquer (ex. "Emplacement inconnu → ignoré") ».  
   - **Problème :** Aucun message explicite n’indique que les emplacements/fournisseurs inconnus sont ignorés.  
   - **Correction :** Ajouter une note ou une infobulle sur l’étape de mapping.

### LOW

6. **Template : extension Excel non mentionnée**  
   - **Fichier :** `apps/web/src/app/(app)/import-stocks/page.tsx` ligne 224  
   - **Problème :** Le bouton dit « Télécharger le template CSV » alors que l’import supporte aussi Excel. Le template est bien en CSV ; le libellé est correct mais pourrait préciser « CSV (compatible Excel) » si pertinent.
