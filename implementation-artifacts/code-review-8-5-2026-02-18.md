## Points d'attention (recommandations mineures)

### 1. Attributs `style` inline restants

**Impact:** 🟡 **Mineur** (non bloquant)

Il reste quelques attributs `style` inline dans les fichiers HTML. Bien que cela ne soit pas explicitement interdit par la Story 8.5 (qui ciblait les balises `<style>`), certains pourraient être migrés vers des classes CSS pour une meilleure cohérence.

**Exemples trouvés:**

```html
<!-- Dans import-stocks.html, import-sales.html, formulas.html, sales.html -->
<input type="text" id="token" style="width: 100%;" />

<!-- Dans stats.html -->
<input type="text" id="tokenInput" style="margin-bottom: 1rem;" />

<!-- Dans forecast.html, custom-formulas.html -->
<div style="margin-top: 0.75rem;">...</div>
<h2 style="margin-top: 1.5rem;">...</h2>
```

**Recommandation:**
- Les `style="display: none;"` sont acceptables car gérés dynamiquement par JavaScript
- Les `style="width: 100%;"` pourraient être remplacés par une classe `.w-full` (si Tailwind est disponible) ou `.full-width` dans `design-system.css`
- Les marges inline (`margin-top`, `margin-bottom`) pourraient être remplacées par des classes utilitaires

**Action suggérée (optionnelle):**
```css
/* Ajouter dans design-system.css */
.full-width { width: 100%; }
.mt-3 { margin-top: 0.75rem; }
.mt-6 { margin-top: 1.5rem; }
.mb-4 { margin-bottom: 1rem; }
```

### 2. Styles inline générés dynamiquement par JavaScript

**Impact:** 🟢 **Acceptable**

Certains styles inline sont générés dynamiquement par JavaScript (ex: `style="background: ${color};"` dans `forecast.html` ligne 432). C'est acceptable car ils dépendent de données dynamiques.

**Exemples:**
```javascript

// forecast.html ligne 452
html += '<li class="comparison-item" style="border-left-color: ' + color + ';">';
```

**Recommandation:** ✅ **Aucune action requise** - Ces cas sont légitimes pour des styles dynamiques.
