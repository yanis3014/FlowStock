## Points d'attention (recommandations)

### 1. Hydration / flash layout (isMobile)

**Impact:** 🟡 **Mineur**

`AppShell` utilise `useEffect` pour détecter `isMobile` (window.innerWidth). Au premier rendu SSR, `isMobile` est `false`, puis se met à jour côté client. Sur mobile, cela peut provoquer un léger flash (sidebar desktop → drawer).

**Recommandation :**
```tsx
// Option : éviter le flash en masquant le shell jusqu'au mount
const [mounted, setMounted] = useState(false);
useEffect(() => setMounted(true), []);
if (!mounted) return <div className="flex h-screen items-center justify-center">Chargement...</div>;
```

Ou utiliser un breakpoint CSS (ex. `lg:hidden` / `hidden lg:block`) plutôt qu'un état JS quand c’est possible.

### 2. Accessibilité

**Impact:** 🟡 **Mineur**

- Overlay mobile a `aria-hidden="true"` ✅
- Le bouton burger n’a pas `aria-expanded` pour indiquer l’état ouvert/fermé
- Pas de piège au focus dans le drawer (touche Escape pour fermer)

**Recommandations :**
- Ajouter `aria-expanded={mobileMenuOpen}` sur le bouton burger
- Gérer `Escape` pour fermer le drawer mobile
- Éventuellement `aria-controls` pour lier le bouton au drawer

### 3. Icônes en ligne

**Impact:** 🟢 **Acceptable**

Les icônes (chevron, burger, close) sont des SVG inline. Pour une maintenance plus simple, envisager `lucide-react` ou un composant `Icon` partagé (prévu dans les stories suivantes ?).

### 4. Gestion d’état Sidebar persistante

**Impact:** 🟢 **Optionnel**

L’état `sidebarCollapsed` n’est pas persisté. À chaque rechargement, la sidebar repart en mode étendu. Si c’est voulu, aucun changement. Sinon, stocker dans `localStorage`.
