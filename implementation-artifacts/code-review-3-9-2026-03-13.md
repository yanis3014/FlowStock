## Problèmes trouvés et corrigés

### 🔴 CRITIQUE — Corrigé

**[C-1] Corruption d'encodage UTF-8 (mojibake) dans `CustomFormulasContent.tsx`**  
Dizaines de chaînes garbled dans l'UI (`뿯½` à la place de caractères accentués français).  
**Fix :** Réécriture complète du fichier avec encodage UTF-8 correct.

---

### 🟠 HAUT — Corrigés

**[H-1] Validation syntaxique en temps réel absente (AC 3)**  
Aucun appel à `/formulas/custom/validate` pendant la frappe.  
**Fix :** Ajout d'un debounce 500ms sur l'expression appelant le endpoint validate. Le textarea affiche un indicateur visuel (✓ valide / ✗ invalide) et les boutons Prévisualiser/Sauvegarder sont désactivés si la syntaxe est invalide.

**[H-2] Bouton "Dupliquer" absent (AC 5)**  
La bibliothèque ne proposait que Modifier et Supprimer.  
**Fix :** Ajout du handler `handleDuplicate` et du bouton "Dupliquer" dans chaque carte de la bibliothèque (crée une copie suffixée "(copie)").

**[H-3] Date de création absente dans la bibliothèque (AC 5)**  
AC 5 exige "liste avec nom, description, date de création".  
**Fix :** Ajout de `created_at` dans `FormulaRow`, `Formula` et les queries SQL (`listCustomFormulas`, `getCustomFormulaById`). Affichage formaté en `fr-FR` dans chaque carte.

**[H-4] Aucun indicateur de position d'erreur (AC 3)**  
AC 3 : "l'utilisateur voit la position approximative de l'erreur".  
**Fix :** Ajout de `error_position?: number` dans `ValidationResult`, extraction de la position depuis les messages mathjs (`char N`/`column N`). Le frontend l'affiche sous le textarea.

---

### 🟡 MOYEN — Corrigés

**[M-1] Erreurs silencieuses dans l'exécution multi-produits**  
Les produits en échec retournaient silencieusement `0`.  
**Fix :** Séparation résultats/erreurs avec `errors?: Record<string, string>` dans `FormulaExecuteResult`. Le frontend affiche les produits en erreur distinctement dans le bloc prévisualisation.

**[M-2] `confirm()` natif pour la suppression**  
**Fix :** Remplacement par une confirmation inline avec boutons "Oui, supprimer" / "Annuler" via state `confirmDeleteId`.

**[M-3] `packages/shared/*` non implémenté**  
Types `CustomFormula` dupliqués entre frontend et backend.  
**Statut :** Non corrigé dans cette review (refactoring packages/shared hors scope immédiat). Action item laissée pour une future story technique.

**[M-4] Fichiers debug dans la racine du projet**  
`custom_formulas_old.txt` (41 Ko) et `formulas_old.txt` (29 Ko).  
**Fix :** Fichiers supprimés.

**[M-5] Cleanup `afterAll` non isolé dans les tests**  
Les DELETE sans filtre tenant pouvaient interférer avec d'autres suites de tests.  
**Fix :** Toutes les DELETE sont maintenant scopées via `WHERE tenant_id IN (SELECT id FROM tenants WHERE slug LIKE 'custom-formula%')`.

---

### 🟢 BAS — Non corrigés (action items)

**[L-1] `listProducts(1000)` codée en dur — troncature silencieuse**  
Pour les tenants avec > 1000 produits, l'exécution multi-produits est incomplète sans avertissement. À adresser dans une story dédiée (pagination de formules multi-produits).

Audit difficile. Les futures stories doivent lister les fichiers explicitement.
